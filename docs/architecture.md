# 量化交易系统架构文档 (Architecture)

> 本文档描述 quant-mft 的整体系统架构、模块职责和数据流向。
> 对应思维导图的完整展开版本，每个模块标注了优先级和实现状态。

---

## 系统全景图

```
📌 量化交易系统架构 (Quant Trading System Architecture)
│
├─ 📂 1. 多源数据接入 (Data Ingestion) ─────── 解决"喂什么数据"
│  ├─ 📈 标准交易执行数据 [P0] 🟢 已实现
│  │  └─ akshare → A 股全市场日线 OHLCV（前复权）
│  │     文件: src/data/fetcher.py
│  ├─ 📣 网络舆情实时信息 [P2] 🔴 远期规划
│  │  └─ 新闻/社群/专业论坛 NLP 情感打分
│  │     文件: src/data/sentiment.py（待创建）
│  ├─ 🪙 数字资产交易数据 [P2] 🔴 远期规划
│  │  └─ ccxt → Binance L2/L3 订单簿深度、大单流
│  │     文件: src/data/crypto_fetcher.py（待创建）
│  └─ 🌐 物理世界替代数据 [P2] 🔴 远期规划
│     └─ ⚠️ 港口吞吐量/交割仓库存/供应链数据
│        注意：数据获取成本极高、清洗难度大，个人开发者慎入
│
├─ 📂 2. 数据清洗与预处理 ─────────────────── 解决"数据质量与噪音控制"
│  ├─ 🛠️ 清洗操作 [P0] 🟡 规划中
│  │  ├─ Winsorization 去极值（用 MAD 而非 3σ，金融数据有 fat tail）
│  │  ├─ 缺失值填充（区分"停牌"和"数据缺失"两种缺失）
│  │  ├─ 时序对齐（所有股票对齐到同一交易日历）
│  │  ├─ 停牌处理（禁止前向填充制造假信号）
│  │  └─ 复权一致性检查（前复权/后复权不能混用）
│  │     文件: src/data/cleaner.py
│  ├─ 📊 信息量度指标 [P1] 🟡 规划中
│  │  ├─ 信息熵 (Information Entropy)：衡量特征包含的信息量
│  │  ├─ 互信息量 (Mutual Information)：特征与目标变量的关联度
│  │  └─ 🛡️ 防火墙逻辑：剔除低信息量特征，从源头控制维度灾难
│  │     文件: src/features/information_metrics.py
│  └─ 🗄️ 数据存储 [P0] 🟢 已实现
│     ├─ 格式：pandas DataFrame → Parquet/CSV 文件
│     ├─ 选型说明：数据量约百万级行，Parquet 完全够用
│     └─ ⚠️ 不需要 ClickHouse（那是几十亿行 tick 数据才需要的）
│         存储路径: data/
│
├─ 📂 3. 特征工程与模型构建 ───────────────── 解决"信号生成与复杂度防范"
│  ├─ 🧩 Alpha 因子提取 [P0] 🟡 规划中
│  │  ├─ 量价因子：收益率（1/5/10/20日）、振幅、换手率变化
│  │  ├─ 动量因子：均线偏离（MA5/10/20/60）、MACD
│  │  ├─ 波动率因子：历史波动率、ATR、布林带宽度
│  │  ├─ 微观结构因子：成交量变化率、量价背离、VWAP偏离
│  │  └─ 目标：构造 20+ 个 Alpha 因子
│  │     文件: src/features/alpha_factors.py
│  ├─ 🎯 因子筛选与降维 [P1] 🟡 规划中
│  │  ├─ IC 分析（Information Coefficient）：因子与未来收益的相关性
│  │  ├─ 因子互相关性矩阵：剔除冗余因子（相关系数 > 0.8）
│  │  └─ PCA 降维：保留 90% 方差的最小维度
│  │     文件: src/features/selector.py
│  ├─ 🤖 模型训练 [P0] 🟡 规划中
│  │  ├─ 🟢 线性模型（强限制）：Ridge/Lasso 缩减估计 [P1]
│  │  ├─ 🟡 非线性模型（限制复杂度）：LightGBM [P0]
│  │  │  ├─ Early Stopping：验证集 loss 不降即停
│  │  │  ├─ L1/L2 正则化：在训练阶段就限制模型复杂度
│  │  │  └─ 树深度/叶子数限制：防止单棵树过拟合
│  │  └─ ⚠️ 关键规则：时间序列必须按时间切分 train/test，禁止随机切！
│  │     文件: src/models/train.py
│  ├─ 📈 模型评估 [P0] 🟡 规划中
│  │  ├─ 分类指标：准确率、AUC、混淆矩阵
│  │  ├─ 复杂度惩罚：AIC/BIC（防止无意义地堆参数）
│  │  ├─ 特征重要性排名（LightGBM 自带 feature_importances_）
│  │  └─ 🧠 心理建设：日线涨跌预测准确率 50-53% 是正常的，不是失败
│  │     文件: src/models/evaluate.py
│  └─ 📝 实验追踪 [P1] 🟡 规划中
│     ├─ 记录每次实验：特征组合、超参、训练日期范围、评估指标
│     ├─ 方案：简单 CSV 日志（MVP 阶段不需要 MLflow）
│     └─ 目标：一周后还能知道哪组参数跑出了最好的结果
│         文件: src/models/experiment.py
│
├─ 📂 4. 严苛验证与回测沙盒 ───────────────── 解决"实战前的拟合测试"
│  ├─ 🎯 手写回测引擎 [P0] 🟡 规划中
│  │  ├─ 逐日遍历：每天检查是否有买入/卖出信号
│  │  ├─ 持仓模拟：记录每笔交易的入场价、出场价、持有天数
│  │  ├─ 交易成本：手续费万三 + 印花税千一（仅卖出）+ 滑点 0.1%
│  │  └─ 输出：资金曲线、每笔交易明细、日度持仓快照
│  │     文件: src/backtest/engine.py
│  ├─ 📏 基准对比 [P0] 🟡 规划中
│  │  ├─ 沪深 300 买入持有策略作为 baseline
│  │  └─ ⚠️ 没有基准的回测毫无意义——牛市里所有策略都赚钱
│  │     文件: src/backtest/benchmark.py
│  ├─ 🛡️ 风控模块 [P0] 🟡 规划中
│  │  ├─ 仓位管理：单票最大仓位限制、组合集中度控制
│  │  ├─ 止损逻辑：固定比例止损 + 移动止损
│  │  └─ 组合层面：行业暴露限制、最大同时持仓数 [P1]
│  │     文件: src/backtest/risk.py
│  └─ 🔮 模拟盘 [P2] 🔴 远期规划
│     └─ 引入真实挂单滑点、交易延迟、手续费磨损
│         （开学后用 xtquant/easytrader 接华泰/东财）
│
├─ 📂 5. 绩效诊断与过拟合防火墙 ──────────── 核心逻辑闭环
│  ├─ 🔍 评估指标 [P0] 🟡 规划中
│  │  ├─ 收益类：年化收益率、累计收益
│  │  ├─ 风险类：最大回撤、年化波动率、下行波动率
│  │  ├─ 综合类：夏普比率、卡玛比率（年化收益/最大回撤）
│  │  ├─ 交易类：胜率、盈亏比、平均持仓天数
│  │  └─ 归因类：行业暴露分析、风格因子暴露
│  ├─ ⚠️ 过拟合预警（量化阈值）[P1] 🟡 规划中
│  │  ├─ 夏普 > 3.0 → 🟡 可疑（日线策略极难达到）
│  │  ├─ 夏普 > 4.0 → 🔴 几乎确定过拟合
│  │  ├─ 年化 > 50% 且最大回撤 < 5% → 🔴 不可能三角
│  │  └─ 触发预警 → 启动蒙特卡洛诊断
│  ├─ 🎲 蒙特卡洛参数扰动测试 [P1] 🟡 规划中
│  │  ├─ 关键参数 ±20% 随机扰动，跑 N=1000 次回测
│  │  └─ 扰动后夏普下降 > 50% → 判定为过拟合
│  │     文件: src/backtest/diagnostics.py
│  └─ 🔄 负反馈闭环优化 [P0] 🟡 规划中
│     ├─ 特征侧：强制因子降维，去除贡献度极低的弱因子
│     ├─ 模型侧：增强 L1/L2 正则化惩罚，剪枝决策树，降低模型自由度
│     └─ 流程：回测 → 诊断 → 修复 → 再回测（迭代直到通过）
│
├─ 📂 6. 实盘部署与监控 ──────────────────── 解决"生命周期与自适应管理"
│  ├─ 🚀 每日自动管线 [P0] 🟡 规划中
│  │  ├─ 拉数据 → 清洗 → 特征工程 → 模型预测 → 信号生成
│  │  └─ 文件: src/pipeline/daily_run.py
│  ├─ ⏰ 定时调度 [P0] 🟡 规划中
│  │  └─ cron (WSL2)，每个交易日 15:30 触发
│  ├─ 📡 Telegram Bot 推送 [P0] 🟡 规划中
│  │  └─ 交易信号 + 每日绩效摘要
│  ├─ 🛑 熔断机制 [P1] 🟡 规划中
│  │  ├─ 单日亏损超阈值 → 自动暂停交易
│  │  └─ 连续 N 天跑输基准 → 人工复核
│  │     文件: src/pipeline/monitor.py
│  └─ 🔁 动态循环 [P2] 🔴 远期规划
│     ├─ 实盘产生的交易数据、滑点数据回流至数据管线
│     └─ 概念漂移检测 (Concept Drift Detection)
│
└─ 📂 横切关注点（贯穿所有模块）
   ├─ 🗃️ 配置管理 [P0]
   │  ├─ config/default.yaml：所有可调参数集中管理
   │  └─ .env：API 密钥、Telegram Token（不提交 git）
   ├─ 📋 日志系统 [P0]
   │  └─ logging 模块：结构化日志（时间戳 + 级别 + 模块名 + 消息）
   ├─ ✅ 代码质量 [P1]
   │  ├─ 中文注释：每行代码解释"为什么这么写"
   │  ├─ pre-commit：black 格式化 + isort 排序
   │  └─ pytest：核心模块单元测试
   └─ 🔗 数据/模型版本管理 [P2]
      └─ DVC 或 Git LFS（远期规划）
```

## 数据流向图

```
[akshare API]
     │
     ▼
[src/data/fetcher.py] ──→ data/a_stock_daily.csv
     │
     ▼
[src/data/cleaner.py] ──→ data/a_stock_daily_clean.parquet
     │
     ▼
[src/features/alpha_factors.py] ──→ data/features.parquet
     │
     ├──→ [src/features/selector.py] ──→ 因子筛选/降维
     │
     ▼
[src/models/train.py] ──→ models/model_YYYYMMDD.txt (LightGBM)
     │
     ▼
[src/strategy/signal_generator.py] ──→ data/signals.csv
     │
     ▼
[src/backtest/engine.py] ──→ 回测报告/资金曲线
     │
     ├──→ [src/backtest/diagnostics.py] ──→ 过拟合诊断
     │
     ▼
[src/pipeline/daily_run.py] ──→ Telegram 推送
```
