## 🆕 四个全新横切模块（原思维导图完全没有）

这些不属于上述任何一个阶段，但贯穿全项目。

### A. 配置管理 [P0]

| 为什么缺它不行 | 对应文件 |
|--------------|---------|
| 模型超参、数据路径、交易参数散落在代码各处 → 回测结果不可复现，是量化项目最常见的低级错误 | `config/default.yaml` + `config/local.yaml` |
| API 密钥、Telegram Token 写死在代码里 → 一 push 到 GitHub 就泄露 | `.env`（已在 .gitignore 中） |

### B. 风险管理 [P0]

| 为什么缺它不行 | 对应文件 |
|--------------|---------|
| 回测好 ≠ 实盘好，没有风控的回测报告是废纸 | `src/backtest/risk.py` |
| 单票 100% 仓位翻倍了不代表策略好，代表运气好 | 仓位管理：单票上限 + 组合集中度 |
| 不止损的策略在实盘里活不过一个月 | 止损逻辑：固定比例 + 移动止损 |

### C. 日志系统 [P0]

| 为什么缺它不行 | 说明 |
|--------------|------|
| `print()` 在生产环境里：没有时间戳、没有级别、没法持久化、没法排查问题 | 用 `logging` 模块，输出到文件 + 控制台 |

### D. 代码质量 [P1]

| 为什么需要 | 说明 |
|-----------|------|
| 6 周后你自己也看不懂当时写了什么 | 中文注释规范（已有） |
| 提交的代码格式乱七八糟 | pre-commit：black + isort |
| 改一个函数不知道有没有破坏其他地方 | pytest 单元测试 |
