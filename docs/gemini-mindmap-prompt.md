# Gemini 思维导图生成提示词

将以下内容复制到 Gemini，让它帮你生成一张完整的量化交易系统架构思维导图。

---

## 提示词（直接复制以下全部内容）

```
你是一位量化交易系统架构师。请根据下面的描述，生成一张完整的**量化交易系统架构思维导图**，用树形文本结构输出（类似文件目录树，使用 ├─ └─ │ 符号，配合 emoji 标注每个节点的类型）。

## 思维导图要求

1. **层级结构**：6 大模块，每个模块下 3-5 个子节点
2. **每个节点标注优先级**：用标签 [P0] [P1] [P2] 区分
   - [P0] = MVP 阶段必须实现（6 周内）
   - [P1] = 加分项（有时间就做）
   - [P2] = v2.0 远期规划（开学后）
3. **用 emoji 区分节点性质**：
   - 🟢 = 已实现或正在实现
   - 🟡 = 已规划但未开始
   - 🔴 = 远期规划
   - ⚠️ = 风险点/注意事项
4. **每个叶子节点加一行简短的中文注释**（用 `# 注释` 格式）

## 六大模块内容

### 模块 1：多源数据接入 (Data Ingestion)
- 标准交易数据 (Market Data)：akshare 拉 A 股日线 OHLCV [P0]
- 网络舆情数据 (Sentiment Data)：新闻/社群 NLP 情感打分 [P2]
- 数字资产数据 (Crypto Data)：ccxt 接 Binance L2 订单簿 [P2]
- 物理世界替代数据 (Alternative Data)：港口吞吐量/供应链 [P2]
- ⚠️ 风险提示：替代数据获取成本高、清洗难度大，个人开发者慎入

### 模块 2：数据清洗与预处理 (Data Cleaning)
- 清洗操作：Winsorization 去极值（用 MAD 而非 3σ）、缺失值填充、时序对齐、停牌处理（禁止前向填充制造假信号）、复权一致性检查 [P0]
- 信息量度：信息熵 (Information Entropy) 与互信息量 [P1]
- 防火墙逻辑：剔除低信息量特征，从源头控制维度灾难 [P1]
- 数据存储：pandas + Parquet 格式（MVP 不需要 ClickHouse，数据量仅百万级）[P0]
- 数据字典：每列含义、来源、更新频率文档化 [P0]

### 模块 3：特征工程与模型构建 (Feature Engineering & Modeling)
- Alpha 因子：量价因子、动量因子、微观结构因子、波动率因子（目标 20+）[P0]
- 线性模型 (强限制)：Ridge/Lasso 缩减估计、PCA 因子降维 [P1]
- 非线性模型 (限制复杂度)：LightGBM + Early Stopping + L1/L2 正则化 [P0]
- 时间序列切分：禁止随机切分，必须按时间顺序 Walk-Forward [P0]
- 评估指标：准确率、AUC、混淆矩阵 + AIC/BIC 模型复杂度惩罚 [P0]
- 实验追踪：MLflow 或 CSV 日志记录每次实验的特征组合、超参、性能 [P1]
- 心理建设：日线涨跌预测 50-53% 准确率是正常的，不是失败 [P0]

### 模块 4：严苛验证与回测沙盒 (Validation & Backtest)
- 手写回测引擎：逐日遍历、持仓模拟、买入/卖出信号 [P0]
- 交易成本模拟：手续费万三、印花税千一（卖出）、滑点 0.1% [P0]
- 基准对比：沪深 300 买入持有策略作为 baseline（没有基准的回测无意义）[P0]
- 样本外测试：时间序列滚动窗口交叉验证 (Walk-Forward Cross-Validation) [P0]
- 模拟盘：引入真实挂单滑点、交易延迟、手续费磨损 [P2]

### 模块 5：绩效诊断与过拟合防火墙 (Diagnostics & Anti-Overfitting)
- 评估指标：夏普比率、最大回撤、年化收益、胜率、盈亏比、泛化误差 [P0]
- 过拟合预警阈值（量化标准）：
  - 夏普 > 3.0 → 🟡 可疑
  - 夏普 > 4.0 → 🔴 几乎确定过拟合
  - 年化 > 50% 且回撤 < 5% → 🔴 不可能三角
- 蒙特卡洛参数扰动测试：关键参数 ±20% 扰动后夏普下降 > 50% → 过拟合 [P1]
- 负反馈闭环机制：
  - 特征侧：强制因子降维，去除低贡献度弱因子 [P0]
  - 模型侧：增加 L1/L2 正则化惩罚，或剪枝决策树 [P0]
- ⚠️ 关键认知：第 4 节和第 5 节是迭代循环，不是两步独立操作

### 模块 6：实盘部署与监控 (Deployment & Monitoring)
- 每日自动管线 (daily_run.py)：拉数据 → 特征工程 → 模型预测 → 回测 → 信号生成 [P0]
- 定时调度：cron (WSL2)，每个交易日 15:30 触发 [P0]
- Telegram Bot 推送：交易信号 + 每日绩效摘要 [P0]
- 熔断机制 (Circuit Breaker)：单日亏损超阈值自动暂停；连续 N 天跑输基准人工复核 [P1]
- 绩效跟踪与漂移检测 (Concept Drift Detection) [P2]
- 动态循环：实盘数据回流至数据管线，滚动更新模型 [P2]

## 独立横切模块（不属于上述 6 个模块，但贯穿全局）

### 配置管理 (Configuration Management)
- config/default.yaml：所有可调参数集中管理（模型超参、数据路径、交易参数）[P0]
- 环境变量 (.env)：API 密钥、Telegram Token 等敏感信息 [P0]
- ⚠️ 量化项目大忌：参数散落在代码各处 → 回测结果不可复现

### 风险管理 (Risk Management)
- 仓位管理：单票最大仓位限制、组合集中度控制 [P0]
- 止损逻辑：固定比例止损 + 移动止损 [P0]
- 组合层面风控：行业暴露限制、最大同时持仓数 [P1]

### 日志与监控 (Logging)
- 结构化日志 (logging 模块，不要用 print) [P0]
- 错误告警：数据拉取失败、模型预测异常时 Telegram 通知 [P1]

### 代码质量 (Code Quality)
- pre-commit hooks：代码格式化 (black)、import 排序 (isort) [P1]
- 单元测试：pytest 覆盖核心模块 [P1]
- 中文注释规范：每行代码解释"为什么这么写" [P0]

## 特别要求

1. 输出完整的一棵树，不要遗漏任何我提到的模块和子节点
2. 每个节点后面的注释要具体，不要泛泛而谈
3. 第 4 节和第 5 节之间加一个双向箭头标注 `⟺ 迭代闭环`
4. 在思维导图最后加一行统计：
   ```
   📊 统计: P0 节点 X 个 | P1 节点 Y 个 | P2 节点 Z 个 | 总节点 N 个
   ```
5. 在统计下方加一行：
   ```
   🎯 MVP 交付物（6周内完成）: [列出所有 P0 节点的文件名]
   ```
```

## 使用方式

1. 打开 [Gemini](https://gemini.google.com)（或 Gemini API / AI Studio）
2. 复制上方 ` ``` ` 包裹的完整提示词
3. 粘贴发送
4. 将输出的思维导图保存到 `docs/architecture.md`

如果 Gemini 输出被截断（因为内容很长），发一句"继续"让它补全。
